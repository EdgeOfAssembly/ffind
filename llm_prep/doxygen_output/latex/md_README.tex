\hypertarget{md_README_autotoc_md126}{}\doxysection{\texorpdfstring{What is ffind?}{What is ffind?}}\label{md_README_autotoc_md126}
{\ttfamily ffind} is an ultra-\/fast file search tool that combines the power of {\ttfamily find} and {\ttfamily grep}. It consists of a daemon that maintains an in-\/memory index of your filesystem using Linux inotify for real-\/time updates, and a client that performs instant searches against this index.\hypertarget{md_README_autotoc_md127}{}\doxysection{\texorpdfstring{Why use ffind?}{Why use ffind?}}\label{md_README_autotoc_md127}

\begin{DoxyItemize}
\item {\bfseries{Instant searches}}\+: No more waiting for recursive directory traversal -\/ the index is always in memory
\item {\bfseries{Real-\/time indexing}}\+: The daemon watches your filesystem and updates the index automatically as files are created, modified, or deleted
\item {\bfseries{Parallel content search}}\+: Utilizes all CPU cores for 6x faster content search than ripgrep
\item {\bfseries{Powerful filters}}\+: Search by name, path, content, size, modification time, and file type with glob patterns or regex
\item {\bfseries{Content search}}\+: Search inside files with fixed-\/string or regex patterns
\item {\bfseries{Combined filters}}\+: Mix multiple criteria for precise results
\end{DoxyItemize}\hypertarget{md_README_autotoc_md128}{}\doxysection{\texorpdfstring{Features}{Features}}\label{md_README_autotoc_md128}
‚úÖ {\bfseries{Real-\/time indexing}} with Linux inotify ~\newline
 ‚úÖ {\bfseries{Instant search}} from in-\/memory index ~\newline
 ‚úÖ {\bfseries{Parallel content search}} using thread pool (all CPU cores) ~\newline
 ‚úÖ {\bfseries{Content search}} with fixed-\/string, regex, or glob patterns ~\newline
 ‚úÖ {\bfseries{Context lines}} (grep-\/style -\/A/-\/\+B/-\/C) ~\newline
 ‚úÖ {\bfseries{Multiple filters}} (name, path, type, size, mtime) ~\newline
 ‚úÖ {\bfseries{Multiple root directories}} support ~\newline
 ‚úÖ {\bfseries{SQLite persistence}} for fast startup ~\newline
 ‚úÖ {\bfseries{Colored output}} with auto-\/detection ~\newline
 ‚úÖ {\bfseries{Glob and regex}} support ~\newline
 ‚úÖ {\bfseries{Graceful directory renames}} (no restart needed) ~\newline

{\bfseries{Benchmark Methodology\+:}} Tests use fair cache management methodology\+:
\begin{DoxyItemize}
\item {\ttfamily find}/{\ttfamily grep} run with cold cache (caches flushed before each run when run with proper privileges)
\item {\ttfamily ffind} runs with warm cache (data in daemon\textquotesingle{}s RAM index)
\item Each benchmark runs 3 times, median time reported
\item This represents real-\/world usage\+: {\ttfamily find}/{\ttfamily grep} read from disk, {\ttfamily ffind} serves from memory
\end{DoxyItemize}

{\bfseries{Why ffind is faster\+:}}
\begin{DoxyItemize}
\item {\bfseries{File metadata searches}}\+: No disk I/O -\/ results served instantly from in-\/memory index
\item {\bfseries{Content searches}}\+: Parallel processing using all CPU cores with minimal disk latency
\item {\bfseries{Real-\/time indexing}}\+: Index stays up-\/to-\/date automatically via inotify
\end{DoxyItemize}

{\bfseries{Cache Management with Dedicated Binary\+:}}
\begin{DoxyItemize}
\item ‚úÖ {\bfseries{Cache-\/flush utility}}\+: Minimal C binary requiring {\ttfamily sudo} for secure cache clearing
\item ‚úÖ {\bfseries{Security best practice}}\+: Only the tiny {\ttfamily cache-\/flush} binary runs with elevated privileges
\item ‚úÖ {\bfseries{Minimal sudo usage}}\+: Only cache-\/flush requires sudo, benchmark scripts run as normal user
\item ‚úÖ {\bfseries{Fair comparison}}\+: {\ttfamily find}/{\ttfamily grep} run with cold cache (disk reads), {\ttfamily ffind} with warm cache (RAM)
\end{DoxyItemize}

{\bfseries{Security Model\+:}}
\begin{DoxyItemize}
\item The {\ttfamily cache-\/flush} binary must run with {\ttfamily sudo} to write to {\ttfamily /proc/sys/vm/drop\+\_\+caches}
\item Only this tiny, auditable C program (\texorpdfstring{$\sim$}{\string~}50 lines) needs elevated privileges
\item All other components (ffind-\/daemon, ffind client, benchmark scripts) run as normal user
\end{DoxyItemize}

{\bfseries{Benefits\+:}}
\begin{DoxyItemize}
\item {\bfseries{Fast startup}}\+: Loads index from database instead of full filesystem scan
\item {\bfseries{Crash-\/safe}}\+: Atomic writes with WAL mode ensure database consistency
\item {\bfseries{Automatic reconciliation}}\+: Detects filesystem changes between runs
\item {\bfseries{Periodic sync}}\+: Database updated every 30 seconds or after 100 changes
\end{DoxyItemize}

{\bfseries{How it works\+:}}
\begin{DoxyEnumerate}
\item On first run, creates SQLite database and indexes filesystem
\item Changes are tracked in memory and periodically flushed to database
\item On restart, loads entries from database and reconciles with actual filesystem
\item Graceful shutdown ensures all pending changes are written
\end{DoxyEnumerate}

{\bfseries{Notes\+:}}
\begin{DoxyItemize}
\item Database file is created if it doesn\textquotesingle{}t exist
\item Database uses WAL (Write-\/\+Ahead Logging) mode for better performance and safety
\item Parent directories must exist for the database path
\item If root paths change, full reconciliation is triggered automatically
\end{DoxyItemize}\hypertarget{md_README_autotoc_md143}{}\doxysubsection{\texorpdfstring{Multiple Root Directories}{Multiple Root Directories}}\label{md_README_autotoc_md143}

\hypertarget{md_README_autotoc_md158}{}\doxysection{\texorpdfstring{FAQ}{FAQ}}\label{md_README_autotoc_md158}
\hypertarget{md_README_autotoc_md159}{}\doxysubsection{\texorpdfstring{Q\+: How fast is it?}{Q\+: How fast is it?}}\label{md_README_autotoc_md159}
{\bfseries{A\+:}} For indexed searches, ffind is nearly instantaneous. Once the daemon has indexed your directories, searches complete in milliseconds regardless of directory size. Traditional tools like {\ttfamily find} need to traverse the filesystem on every search, which can take seconds or minutes on large directory trees.

The initial indexing time depends on the size of your directory tree. On modern hardware\+:
\begin{DoxyItemize}
\item Small directories (\texorpdfstring{$<$}{<} 10,000 files)\+: \texorpdfstring{$<$}{<} 1 second
\item Medium directories (10,000 -\/ 100,000 files)\+: 1-\/5 seconds ~\newline

\item Large directories (100,000+ files)\+: 5-\/30 seconds
\end{DoxyItemize}

After initial indexing, the daemon maintains the index in real-\/time with negligible overhead.\hypertarget{md_README_autotoc_md160}{}\doxysubsection{\texorpdfstring{Q\+: Does it use a lot of RAM?}{Q\+: Does it use a lot of RAM?}}\label{md_README_autotoc_md160}
{\bfseries{A\+:}} Memory usage is proportional to the number of indexed files. On average\+:
\begin{DoxyItemize}
\item Small directories (\texorpdfstring{$<$}{<} 10,000 files)\+: \texorpdfstring{$<$}{<} 10 MB
\item Medium directories (10,000 -\/ 100,000 files)\+: 10-\/50 MB
\item Large directories (100,000+ files)\+: 50-\/200 MB
\end{DoxyItemize}

Each indexed entry stores the file path, size, modification time, and metadata. For most use cases, memory usage is minimal compared to modern system RAM.\hypertarget{md_README_autotoc_md161}{}\doxysubsection{\texorpdfstring{Q\+: What about huge directory trees?}{Q\+: What about huge directory trees?}}\label{md_README_autotoc_md161}
{\bfseries{A\+:}} ffind handles large directory trees efficiently\+:


\begin{DoxyEnumerate}
\item {\bfseries{Indexing}}\+: Initial indexing shows progress every 10,000 entries (in foreground mode)
\item {\bfseries{Memory}}\+: Uses efficient C++ data structures to minimize memory overhead
\item {\bfseries{Persistence}}\+: Use {\ttfamily -\/-\/\doxylink{ffind-daemon_8cpp_ad6e663497d2c934364b3bcf07496b30b}{db}} option to save the index to SQLite for fast startup
\item {\bfseries{Multiple roots}}\+: Index only the directories you need, not the entire filesystem
\end{DoxyEnumerate}

For very large trees (\texorpdfstring{$>$}{>} 1 million files), consider\+:
\begin{DoxyItemize}
\item Using SQLite persistence ({\ttfamily -\/-\/\doxylink{ffind-daemon_8cpp_ad6e663497d2c934364b3bcf07496b30b}{db}}) for faster restarts
\item Splitting into multiple daemons with different roots
\item Excluding large binary directories you don\textquotesingle{}t need to search
\end{DoxyItemize}\hypertarget{md_README_autotoc_md162}{}\doxysubsection{\texorpdfstring{Q\+: How does persistence work?}{Q\+: How does persistence work?}}\label{md_README_autotoc_md162}
{\bfseries{A\+:}} When you enable SQLite persistence with {\ttfamily -\/-\/\doxylink{ffind-daemon_8cpp_ad6e663497d2c934364b3bcf07496b30b}{db} /path/to/db}\+:


\begin{DoxyEnumerate}
\item {\bfseries{First run}}\+: The daemon indexes your filesystem and saves entries to the database
\item {\bfseries{Subsequent runs}}\+: The daemon loads the index from the database (much faster than re-\/scanning)
\item {\bfseries{Reconciliation}}\+: The daemon automatically detects and handles changes since last run
\item {\bfseries{Updates}}\+: Changes are batched and saved periodically (every 30 seconds or 100 changes)
\item {\bfseries{Crash safety}}\+: Uses WAL (Write-\/\+Ahead Logging) mode for atomic writes
\end{DoxyEnumerate}

This means\+:
\begin{DoxyItemize}
\item ‚úÖ Fast startup even after system reboot
\item ‚úÖ No data loss on crashes or power failures ~\newline

\item ‚úÖ Automatic sync between database and filesystem
\end{DoxyItemize}\hypertarget{md_README_autotoc_md163}{}\doxysubsection{\texorpdfstring{Q\+: Can I use it on network filesystems (NFS, CIFS)?}{Q\+: Can I use it on network filesystems (NFS, CIFS)?}}\label{md_README_autotoc_md163}
{\bfseries{A\+:}} inotify (which ffind uses for real-\/time monitoring) only works on local filesystems. For network filesystems\+:
\begin{DoxyItemize}
\item ‚ùå Real-\/time monitoring won\textquotesingle{}t work
\item ‚úÖ You can still use ffind, but you\textquotesingle{}ll need to restart the daemon to pick up changes
\item üí° Consider using {\ttfamily locate} or scheduled rescans for network shares
\end{DoxyItemize}\hypertarget{md_README_autotoc_md164}{}\doxysubsection{\texorpdfstring{Q\+: How do I search multiple directories?}{Q\+: How do I search multiple directories?}}\label{md_README_autotoc_md164}

The client will search across all monitored directories automatically.\hypertarget{md_README_autotoc_md165}{}\doxysubsection{\texorpdfstring{Q\+: What\textquotesingle{}s the difference between {\ttfamily -\/c} and {\ttfamily -\/g}?}{Q\+: What\textquotesingle{}s the difference between {\ttfamily -\/c} and {\ttfamily -\/g}?}}\label{md_README_autotoc_md165}
{\bfseries{A\+:}} Both search file contents, but differently\+:
\begin{DoxyItemize}
\item {\ttfamily -\/c "{}TODO"{}}\+: Fixed-\/string substring search (fastest)
