Fast daemon-\/based file finder with real-\/time inotify indexing.\hypertarget{md_README_autotoc_md11}{}\doxysection{\texorpdfstring{What is ffind?}{What is ffind?}}\label{md_README_autotoc_md11}
{\ttfamily ffind} is an ultra-\/fast file search tool that combines the power of {\ttfamily find} and {\ttfamily grep}. It consists of a daemon that maintains an in-\/memory index of your filesystem using Linux inotify for real-\/time updates, and a client that performs instant searches against this index.\hypertarget{md_README_autotoc_md12}{}\doxysection{\texorpdfstring{Why use ffind?}{Why use ffind?}}\label{md_README_autotoc_md12}

\begin{DoxyItemize}
\item {\bfseries{Instant searches}}\+: No more waiting for recursive directory traversal -\/ the index is always in memory
\item {\bfseries{Real-\/time indexing}}\+: The daemon watches your filesystem and updates the index automatically as files are created, modified, or deleted
\item {\bfseries{Powerful filters}}\+: Search by name, path, content, size, modification time, and file type with glob patterns or regex
\item {\bfseries{Content search}}\+: Search inside files with fixed-\/string or regex patterns
\item {\bfseries{Combined filters}}\+: Mix multiple criteria for precise results
\end{DoxyItemize}\hypertarget{md_README_autotoc_md13}{}\doxysection{\texorpdfstring{Requirements}{Requirements}}\label{md_README_autotoc_md13}

\begin{DoxyItemize}
\item Linux (uses inotify for filesystem monitoring)
\item g++ with C++20 support
\item pthread library
\end{DoxyItemize}\hypertarget{md_README_autotoc_md14}{}\doxysection{\texorpdfstring{Build}{Build}}\label{md_README_autotoc_md14}

{\bfseries{Note}}\+: {\ttfamily -\/g} is mutually exclusive with {\ttfamily -\/c} and {\ttfamily -\/r}. Use {\ttfamily -\/c} for fixed string search, {\ttfamily -\/c} with {\ttfamily -\/r} for regex search, or {\ttfamily -\/g} for glob pattern search.\hypertarget{md_README_autotoc_md26}{}\doxysubsection{\texorpdfstring{Context lines}{Context lines}}\label{md_README_autotoc_md26}
Context lines work just like grep\textquotesingle{}s {\ttfamily -\/A}, {\ttfamily -\/B}, and {\ttfamily -\/C} options, showing surrounding lines for better context when searching file contents. These flags require {\ttfamily -\/c} or {\ttfamily -\/g} (content search).


\begin{DoxyItemize}
\item {\ttfamily -\/A N} -\/ Show N lines {\bfseries{after}} each match
\item {\ttfamily -\/B N} -\/ Show N lines {\bfseries{before}} each match
\item {\ttfamily -\/C N} -\/ Show N lines {\bfseries{before and after}} each match (shorthand for {\ttfamily -\/A N -\/B N})
\end{DoxyItemize}\hypertarget{md_README_autotoc_md27}{}\doxysubsubsection{\texorpdfstring{Output format}{Output format}}\label{md_README_autotoc_md27}
Context lines use the same grep-\/style format\+:
\begin{DoxyItemize}
\item {\bfseries{Match lines}}\+: {\ttfamily path\+:lineno\+:content} (colon before content)
\item {\bfseries{Context lines}}\+: {\ttfamily path\+:lineno-\/content} (dash before content)
\item {\bfseries{Group separator}}\+: {\ttfamily -\/-\/} appears between non-\/contiguous match groups
\end{DoxyItemize}

When match contexts overlap, they are automatically merged into a single group without separators.

\hypertarget{md_README_autotoc_md28}{}\doxysubsection{\texorpdfstring{Color output}{Color output}}\label{md_README_autotoc_md28}
The {\ttfamily -\/-\/color} option controls colored output for better readability\+:


\begin{DoxyItemize}
\item {\ttfamily -\/-\/color=auto} (default) -\/ Automatically enables colors when output is to a terminal
\item {\ttfamily -\/-\/color=always} -\/ Forces colored output even when piped or redirected
\item {\ttfamily -\/-\/color=never} -\/ Disables all colored output (useful for scripts and piping)
\end{DoxyItemize}

When colors are enabled\+:
\begin{DoxyItemize}
\item {\bfseries{File paths}} are displayed in bold
\item {\bfseries{Line numbers}} (in content search) are shown in cyan
\item {\bfseries{Matched content}} is highlighted in bold red
\end{DoxyItemize}
