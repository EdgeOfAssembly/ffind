\doxysubsection{Detailed Description}
Class\+: \doxylink{structMappedFile}{Mapped\+File} Purpose\+: RAII wrapper for memory-\/mapped files used in content search

Security Considerations\+:
\begin{DoxyItemize}
\item Uses MAP\+\_\+\+PRIVATE to prevent modifications from affecting original file
\item Handles errors gracefully (returns invalid on any failure)
\item Empty files are handled correctly (valid but no data)
\item madvise() failure is non-\/fatal (advisory only)
\end{DoxyItemize}

Memory Safety\+:
\begin{DoxyItemize}
\item RAII\+: Automatically unmaps and closes on destruction
\item Copy-\/prevention\+: Deleted copy constructor and assignment
\item Move semantics\+: Not implemented (each thread creates its own)
\end{DoxyItemize}

Thread-\/safety\+: Not thread-\/safe (each thread creates its own instance)

Implementation Notes\+:
\begin{DoxyItemize}
\item Uses mmap() for efficient file reading without copying to userspace
\item MADV\+\_\+\+SEQUENTIAL hints kernel for better readahead
\item Fallback gracefully if mmap() fails (returns invalid)
\end{DoxyItemize}

REVIEWER\+\_\+\+NOTE\+: This is the core of efficient content search. By using mmap(), we avoid copying file contents into userspace buffers. The kernel provides the pages directly from its page cache.
